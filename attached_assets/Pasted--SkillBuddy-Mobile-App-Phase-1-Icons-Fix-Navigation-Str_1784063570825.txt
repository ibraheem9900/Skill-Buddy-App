## TASK 1 — CRITICAL: Fix Icons Not Rendering on Device (iOS & Android)

**Problem:** Icons are currently not showing up at all when running the app on a real phone (both iOS and Android need to be confirmed working).

**Required fix — diagnose and fix properly:**
- If using `lucide-react-native` or `@expo/vector-icons`, confirm the icon font is actually being loaded before first render. For `@expo/vector-icons`, fonts are usually bundled automatically with Expo, but if using `lucide-react-native` or another SVG-based icon set, confirm the package is correctly installed and that no font-loading step (`expo-font` / `useFonts`) is missing — SVG-based icon libraries like `lucide-react-native` don't need font loading at all, so if icons are missing there, check for incorrect import paths, wrong prop names (e.g. `size`/`color` vs `width`/`height`), or a version mismatch with the installed React Native/Expo SDK version.
- Check that this isn't a Metro bundler cache issue — try a clean start (`expo start -c`) to rule out stale bundling.
- Check that any icon component isn't accidentally rendering with `color="transparent"`, `opacity: 0`, or zero `size` inherited from a broken theme-token reference (especially if this broke around the same time as the dark mode theme-token work from a previous phase — check whether icon `color` props are pulling from a theme token that's now undefined/broken).
- Verify the fix by confirming icons render correctly in a physical device build or Expo Go on both an iOS and an Android device/simulator — not just in a web preview, since icon rendering issues often differ between web and native.
- This is blocking essentially all other visual work, so treat it as the highest priority item in this pass.
